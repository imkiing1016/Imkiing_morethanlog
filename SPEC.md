# 멀티플레이어 블러핑 주식게임 — 개발 지침서 (SPEC)

> 한 줄 정의: 매 회차, 내가 아는 미래 정보를 **진실 혹은 뻥카로 흘려** 남을 내 주식에 끌어들이고, 동시에 남의 뻥카를 간파해 차익을 먹는 링크 공유형 멀티플레이어 게임.

이 문서는 Claude Code가 단독으로 구현을 진행할 수 있도록 작성된 단일 진실 원천(Single Source of Truth)이다. 모든 구현 판단은 이 문서를 우선한다. 문서와 코드가 충돌하면 이 문서를 따르고, 변경이 필요하면 먼저 이 문서를 수정한 뒤 코드를 고친다.

---

## 0. 기술 스택 (고정)

| 레이어 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | Next.js (App Router) + TypeScript | 기존 환경 유지 |
| 웹 배포 | Vercel | 정적/SSR 웹 |
| 실시간 동기화 | Node WebSocket 서버 (`ws`) | 방(room) 단위 상태. 게임 1판 = 1 room. 호스팅은 Render(브라우저 GitHub 배포) |
| 권위 서버 | 위 WebSocket 서버 (in-memory `GameRoom`) | **모든 게임 상태 계산은 서버에서만** |
| 영속 DB | Supabase Postgres | 방/판/결과 기록 (선택, MVP에선 in-memory로 시작 가능) |
| 상태관리(클라) | Zustand | 서버 스냅샷 수신 → 렌더 |
| 클라 소켓 | partysocket (재연결 래퍼) | 일반 WebSocket. 서버와 무관하게 재연결만 담당 |
| 스타일 | Tailwind CSS | 디자인 토큰은 5장 참고 |

> 실시간 계층은 원래 PartyKit이었으나, 서비스 불안정으로 **자체 호스팅 Node WebSocket 서버**로 교체했다(서버 권위 원칙은 동일). 방 상태는 서버 메모리의 `GameRoom` 인스턴스(roomCode 키)로 보관한다.

**핵심 원칙: 클라이언트는 입력만 보내고, 서버가 상태를 계산해 모든 방 인원에게 브로드캐스트한다.** 주가/거래/이벤트/정산을 클라가 계산하면 절대 안 된다(치팅 방지). 클라는 서버가 내려준 스냅샷을 그릴 뿐이다.

## 1. 게임 개요

- 플레이어 수: 3~6명 (권장 4명)
- 한 판: 8회차 (설정으로 6/8/10 선택 가능)
- 승리 조건: 마지막 회차 종료 시 **총자산 최다**
  - 총자산 = 보유 현금 + (보유 주식 평가액 합) + (내 회사 미보유 지분 평가액)
- 진행 방식: **동시 턴(보드게임형)**. 모든 플레이어가 각 페이즈를 동시에 입력하고 일괄 정산. 거래 페이즈만 제한시간 내 실시간 호가 허용(하이브리드).

각 플레이어는 **회사 하나 = 주식 하나**다. 플레이어는 자기 회사를 운영하면서 동시에 남의 주식을 사고판다.

### 1.1 게임 시작 — 사업 설립 (SETUP 페이즈)
- 호스트가 시작하면 로비 → **SETUP** 페이즈로 간다. 각 플레이어는 자기 사업을 만든다:
  - **카테고리 1개 선택** (6종): `IT_GAME(IT/게임)`, `BEAUTY(뷰티)`, `CONSTRUCTION(건설)`, `RETAIL(유통)`, `BIO(바이오)`, `DEFENSE(방산)`
  - **회사명 직접 입력** (최대 20자)
- 모든 플레이어의 **시작 시총은 동일**하다: `startingPrice × sharesOutstanding`(전원 같은 값). 시작 신뢰도/기술 레벨도 동일.
- 카테고리 중복 허용(서로 다른 플레이어가 같은 카테고리 가능).
- 연결된 전원이 사업 설립을 마치면 1회차 정보 페이즈로 넘어간다.

## 2. 코어 루프 (1회차 = 5단계)

페이즈는 반드시 이 순서. 이 순서가 게임의 생명이다(포지션을 선언보다 먼저 깔아야 블러핑이 손익으로 연결됨).

### ① 정보 페이즈 (비공개)
- 각 플레이어는 자기 섹터의 **다음 회차 이벤트 방향**만 받는다: `BULLISH(호재)` 또는 `BEARISH(악재)`.
- **강도(magnitude)는 비공개이며 랜덤.** 방향만 알고 크기는 모른다. (정보 우위 100% 방지 장치)
- 이 정보는 본인에게만 보인다.

### ② 사전 포지션 페이즈 (비공개)
- 선언하기 **전에**, 자기 자본 내에서 매수/매도를 비공개로 확정한다.
- 자기 회사 주식, 남의 회사 주식 모두 거래 가능.
- 이 포지션은 다른 플레이어에게 보이지 않는다. (선언이 미끼로 작동하기 위한 핵심)
- 제약: 자기 회사 지분은 최대 60%까지만 보유 가능(독식 방지).

### ③ 선언 페이즈 (공개)
- 각자 **전망 카드 1장**을 공개로 낸다:
  - `HYPE(떡상 예고)` / `WARN(위기 경고)` / `SILENT(침묵)`
- 선언은 자유. 실제 정보와 일치할 필요 없음(= 블러핑 가능).
- 진실 여부는 ④ 정산에서 공개된다.

### ④ 거래 페이즈 (공개, 제한시간)
- 모든 플레이어의 선언이 공개된 상태.
- 제한시간(기본 45초) 동안 누구 주식이든 사고판다.
- 각 회사 카드에 **선언 배지 + 신뢰도(별점)** 를 함께 노출 → 의심의 근거 제공.
- 가격은 이 페이즈에서 **거래량에 따라 즉시 변동**한다(매수 압력 ↑ → 주가 ↑).

### ⑤ 정산 페이즈
- 다음 회차로 예고됐던 실제 이벤트가 **이번 회차 종료 시 발동**한다. (정보 페이즈에서 "다음 회차"라 부른 이벤트가 여기서 실현)
- 실제 이벤트 방향 + 랜덤 강도로 주가 +/- 적용.
- 각 플레이어의 선언이 실제 정보와 일치했는지 공개:
  - 선언 == 실제 → **신뢰도 +1** (단 SILENT는 변동 없음)
  - 선언 != 실제 → **신뢰도 −1**
- 손익 정산 후 다음 회차로.

### 페이즈 전환 규칙 (서버 권위)
- 모든 페이즈 전환은 **서버만** 결정하고 전체에 브로드캐스트한다. 클라는 페이즈를 바꿀 수 없고, 입력 완료 시 **"준비됨(ready)" 신호**만 보낸다.
- 순서는 위 ①~⑤ 고정. 절대 건너뛰지 않는다. ⑤ 정산이 끝나면 `round`가 1 증가하고 다음 회차 ①로 돌아간다. `round`가 `maxRounds`를 넘기면 `ENDED`.
- **조기 전환**: 연결된 모든 플레이어가 해당 페이즈에서 ready 신호를 보내면 타이머 전이라도 즉시 다음 페이즈로 넘어간다. **단 거래(TRADE) 페이즈는 예외** — `phaseDeadline` 타이머가 끝날 때만 서버가 자동 전환한다(실시간 호가를 보장하기 위해).
- 각 페이즈로 진입할 때 플레이어의 `ready`는 모두 false로 초기화된다.
- 페이즈 전환마다 사람이 읽을 한 줄을 `GameState.log`에 남긴다.

## 3. 핵심 메커니즘 상세

### 3.1 신뢰도(Trust) = 화폐
- 범위: 0 ~ 5 (시작값 3). UI는 별 5개로 표현.
- 신뢰도는 **다음 회차 내 선언의 시장 영향력**을 결정한다.
  - 영향력 계수 = `0.4 + 0.12 * trust` (trust 0 → 0.4배, trust 5 → 1.0배)
  - 선언이 거래 페이즈에서 다른 플레이어 주가에 주는 기대 변동폭에 이 계수를 곱한다.
  - 즉 신뢰도 0이면 선언해도 시장이 거의 안 움직임 = 뻥카 무한 방지.
- **설계 의도**: "언제 진실을 쌓아두다, 결정적일 때 한 방 거짓말을 칠까"라는 자원 관리 게임으로 만든다.

### 3.2 블러핑의 4분면 (플레이어가 버는 방식)
실제 정보 × 선언의 교차로 4가지 수익 경로가 성립한다. **선언 전에 포지션을 먼저 깔기 때문에** 모든 칸에서 본인이 직접 이득을 본다.

| 실제 | 선언 | 전략 | 내 이득 경로 |
|---|---|---|---|
| 호재 | WARN(거짓) | 거짓 악재 → 남이 던짐 | 싸게 매집 후 폭등 |
| 악재 | HYPE(거짓) | 거짓 호재 → 남이 매수 | 고점 떠넘기고 현금화 |
| 호재 | HYPE(진실) | 정직 + 선매집 | 따라사기 유도로 평가익 + 신뢰도↑ |
| 악재 | WARN(진실) | 정직 + 선매도 | 폭락 전 회피 + 신뢰도↑ |

### 3.3 스탯 / 기술발전
- 회사는 `techLevel`(1~5)을 가진다. 기초체력(펀더멘털).
- 현금을 투입해 techLevel을 올리면 **이벤트와 무관한 기본 주가 상승률**이 오른다.
- **설계 의도**: 신뢰도 0이어도 techLevel 높으면 주가가 버틴다 → 정직/펀더멘털 루트 보장, 한 전략 독주 방지.
- 업그레이드는 회차 루프 바깥의 "관리 화면"에서 수행(회차당 1회).

### 3.4 사업 전환 (피벗)
- 현재 섹터를 버리고 다른 섹터로 전환.
- 비용: 큼(예: 현재 회사가치의 30%). 효과: **신뢰도 리셋(3으로) + 새 섹터의 정보 획득**.
- "정체성 세탁" 용도의 후반 카드. 관리 화면에서 수행.

### 3.5 엑시트 (회사 전체 매각)
- 자기 회사를 통째로 다른 플레이어에게 매각 → 현금화.
- 매각가 = 시장 평가액 × (0.8 + 0.08 × trust) 의 **최소 호가**. 그 위로는 협상/입찰.
- **신뢰도가 매각가에 직접 연결**된다(신뢰도 일관성).
- 산 사람은 두 회사를 운영하거나 합병. 후반 역전 장치.
- 관리 화면 또는 별도 모달에서 수행.

### 3.6 글로벌 이벤트
- 매 회차 정산 시 **섹터별 글로벌 이벤트** 1개 발동(개인 정보 이벤트와 별개의 시장 전체 이벤트).
- 효과: 특정 섹터 전체에 +/- (예: "바이오 R&D 보조금 확대 → 바이오 +35%").
- 평균회귀 요소 포함: 직전 회차 과열 섹터에 역풍 이벤트 가중치 ↑ → 선두 독주 방지.

## 4. 데이터 모델 (서버 권위 상태)

TypeScript 타입으로 정의. 서버(PartyKit room)가 이 상태를 보유하고 클라에 스냅샷을 내린다.

```ts
// 사업 카테고리 6종 (라벨: IT/게임, 뷰티, 건설, 유통, 바이오, 방산)
type Sector = 'IT_GAME' | 'BEAUTY' | 'CONSTRUCTION' | 'RETAIL' | 'BIO' | 'DEFENSE';
type Phase = 'LOBBY' | 'SETUP' | 'INFO' | 'POSITION' | 'DECLARE' | 'TRADE' | 'SETTLE' | 'MANAGE' | 'ENDED';
type Declaration = 'HYPE' | 'WARN' | 'SILENT';
type Direction = 'BULLISH' | 'BEARISH';

interface Company {
  ownerId: string;        // 플레이어 id
  name: string;
  sector: Sector;
  price: number;          // 현재 주가
  techLevel: number;      // 1~5
  trust: number;          // 0~5
  sharesOutstanding: number;
}

interface PlayerState {
  id: string;
  nickname: string;
  cash: number;
  holdings: Record<string /*companyOwnerId*/, number /*shares*/>;
  // 비공개(본인만): 이번 회차 내 섹터의 다음 이벤트 방향
  privateInfo?: Direction;
  // 비공개: 이번 회차 사전 포지션(체결 전 의도) — 서버만 보관, 정산 시 반영
  pendingPosition?: Array<{ companyOwnerId: string; shares: number /* +매수 -매도 */ }>;
  // 공개: 이번 회차 선언
  declaration?: Declaration;
  // 공개: 이번 페이즈 입력 완료 신호(서버의 조기 전환 판단용). 매 페이즈 진입 시 false.
  ready: boolean;
  connected: boolean;
}

interface GameState {
  roomCode: string;
  phase: Phase;
  round: number;          // 1..maxRounds
  maxRounds: number;
  hostId: string;
  players: PlayerState[];
  companies: Record<string /*ownerId*/, Company>;
  phaseDeadline?: number; // epoch ms, 거래 페이즈 타이머
  // 이번 회차 정산에서 적용될 글로벌 이벤트
  pendingGlobalEvent?: { sector: Sector; magnitude: number; headline: string };
  log: Array<{ round: number; text: string }>;
}
```

서버는 위 상태 중 **비공개 필드(privateInfo, pendingPosition)를 해당 플레이어에게만** 내려보내고, 나머지는 전체 공개. 절대 다른 플레이어의 privateInfo/pendingPosition을 브로드캐스트하지 말 것.

## 5. 디자인 토큰 / UI 규칙

기존에 합의된 화면(①사전포지션 ②선언 ③거래 ④정산)을 기준으로 한다. 모바일 우선(약 380px), 단계별 색 구분.

### 페이즈별 액센트 색
| 페이즈 | 의미 | 색 키 |
|---|---|---|
| INFO / POSITION | 비공개(눈 가림) | danger(빨강) |
| DECLARE | 공개 선언 | warning(노랑) |
| TRADE | 거래/시간제한 | success(초록) |
| SETTLE | 이벤트 정산 | info(파랑) |

### UI 디자인 포인트(반드시 반영)
1. **선언 화면**: 각 선택지(HYPE/WARN/SILENT) 아래 "이걸 고르면 내가 어떻게 버는지" 한 줄 설명. 4분면 전략이 메뉴처럼 읽혀야 함.
2. **거래 화면**: 각 회사 카드에 [선언 배지 + 신뢰도 별점]을 나란히. 의심의 근거를 한눈에.
3. **정산 화면**: 한 줄에 [선언 → 진실/거짓 + 신뢰도 변동 + 손익]을 묶어 블러핑 결과가 통쾌하게 닫히도록.
4. **비공개 페이즈**엔 항상 "비공개 단계" 배지(눈 가림 아이콘)를 상단에 노출 → 지금 내 정보가 안 보인다는 안심.

### 스타일 규칙
- 플랫 디자인. 그라데이션/그림자 금지(포커스 링 제외).
- 라운드: 카드 12px, 요소 8px.
- 폰트 굵기: 400/500 두 가지만.
- 문장은 sentence case. 전부 대문자/타이틀 케이스 금지.
- 색은 의미를 인코딩(호재=초록, 악재=빨강, 중립=회색). 무지개색 순환 금지.
- 숫자는 항상 반올림/통화 포맷으로 출력(부동소수점 노출 금지).

## 6. 구현 단계 (마일스톤)

순서대로 진행. 각 단계는 독립 실행/검증 가능해야 한다.

- **M0 — 스캐폴드**: Next.js + TS + Tailwind + PartyKit 연결. 빈 방 생성/입장 동작.
- **M1 — 로비/방**: 방 코드 생성, 링크 공유, 닉네임 입력, 호스트가 시작. 3~6명 입장.
- **M2 — 상태머신**: 5페이즈 전환을 서버 권위로 구현. 더미 데이터로 한 회차 순환.
- **M3 — 코어 루프**: 정보→포지션→선언→거래→정산 실제 로직. 신뢰도/4분면/주가변동 계산.
- **M4 — UI 4화면**: 5장 토큰대로 모바일 화면 구현, 서버 스냅샷 바인딩.
- **M5 — 관리 화면**: 스탯/피벗/엑시트.
- **M6 — 이벤트/밸런스**: 글로벌 이벤트, 평균회귀, 밸런스 상수 튜닝.
- **M7 — 마감**: 승리화면/리매치/재연결 처리/엣지케이스.

## 7. 밸런스 상수 (한 곳에 모아 튜닝)

```ts
export const BALANCE = {
  startingCash: 8000,
  startingPrice: 1000,
  startingTrust: 3,
  startingTech: 1,
  maxSelfOwnership: 0.60,   // 자기 회사 지분 상한
  trustInfluence: (t: number) => 0.4 + 0.12 * t,
  eventMagnitudeRange: [0.15, 0.40] as const, // 강도 랜덤 범위
  tradeWindowSec: 45,
  techUpgradeCost: (lvl: number) => 1000 * lvl,
  pivotCostRate: 0.30,
  exitMinPriceRate: (t: number) => 0.8 + 0.08 * t,
  meanReversionWeight: 0.5, // 과열 섹터 역풍 가중
};
```

이 값들은 M6에서 플레이테스트로 조정한다. 하드코딩 금지, 반드시 이 객체 참조.

## 8. 반드시 지킬 것 / 하지 말 것

**지킬 것**
- 모든 게임 계산은 서버에서. 클라는 렌더만.
- 비공개 필드는 소유자에게만 전송.
- 밸런스 값은 BALANCE 객체에서만.
- 페이즈 순서 고정(포지션이 선언보다 먼저).

**하지 말 것**
- 클라이언트에서 주가/손익 계산.
- 다른 플레이어의 privateInfo/pendingPosition 노출.
- localStorage/sessionStorage 사용(Artifacts/일부 환경 제약). 상태는 서버+메모리.
- 페이즈를 건너뛰거나 순서를 바꾸기.
